Creating and Using a Class

Classes are the foundation of Object-Oriented Programming (OOP) — a powerful way to model real-world objects and situations in code.

What is a class?

  • A set of instructions for how to create individual instances
  • Defines the general behavior that a category of objects can have
  • Each instance automatically gets this general behavior and can have unique traits

Example: Creating a Lion Class

Scenario: We need a class to represent a lion for a lion’s den - any lion in general.

What do we know about lions?

  • They have a name and an age
  • Common behaviors: roar and pace

These attributes and behaviors will go in our Lion class.

lion.py

class Lion:
    """A simple model of a lion from the den."""
    
    def __init__(self, name, age):
        """Initialize name and age attributes."""
        self.name = name
        self.age = age
    
    def roar(self):
        """Simulate a lion roaring."""
        print(f"{self.name} lets out a mighty roar!")
    
    def pace(self):
        """Simulate a lion pacing in the den."""
        print(f"{self.name} paces back and forth nervously.")

Understanding the Class Structure

1. Class Definition

class Lion:
  • By convention, capitalized names refer to classes in Python
  • No parentheses because we’re creating this class from scratch

2. The __init__() Method

    def __init__(self, name, age):

  • Two underscores on each side prevent naming conflicts
  • Always include self (“instance parameter”) as the first parameter
    def __init__(self, name, age):
        """Initialize name and age attributes."""
        self.name = name
        self.age = age

3. The self Parameter

  • Required and must come first
  • A reference to the instance itself
  • Gives the instance access to attributes and methods in the class
  • Automatically passed when creating or calling methods

4. Instance Attributes

  • Variables with the self prefix: self.name, self.age
  • Accessible through any instance created from the class
  • Called attributes

5. Methods

    def roar(self):
        """Simulate a lion roaring."""
        print(f"{self.name} lets out a mighty roar!")
    
    def pace(self):
        """Simulate a lion pacing in the den."""
        print(f"{self.name} paces back and forth nervously.")
  • roar() and pace() define the lion’s behaviors
  • Each method has self as its only parameter
  • Methods can be called on any instance

💡 Notes:

  • The instance parameter is conventionally named self, but you can choose any name (e.g. this). However, using self is a widely followed convention that makes your code easier to read and understand for other programmers.
  • The double underscores are called dunder (short for “double underscore”) or magic methods. They signal to Python that this method has a special, built-in meaning that Python itself knows about and calls automatically.

Making an Instance from a Class

Now let’s create an instance — a specific lion from the Lion class:

my_lion = Lion('Mufasa', 12)

print(f"My lion's name is {my_lion.name}.")
print(f"My lion is {my_lion.age} years old.")

What happens:

  1. Python reads Lion('Mufasa', 12) and calls __init__() with arguments 'Mufasa' and 12
  2. The __init__() method creates an instance and sets the attributes
  3. Python returns the instance and assigns it to the variable my_lion

Naming convention:

  • Capitalized names (like Lion) refer to classes
  • Lowercase names (like my_lion) refer to instances

Accessing Attributes

Use dot notation to access an instance’s attributes:

my_lion.name  # Access the name attribute
my_lion.age   # Access the age attribute

Python looks at the instance and finds the attribute associated with it.

Calling Methods

Use dot notation to call any method defined in the class:

my_lion.roar()
my_lion.pace()

How it works:

  • Give the instance name (my_lion) and the method name (roar), separated by a dot
  • Python looks for the method in the class and runs that code
  • Descriptive names like roar() and pace() make code easy to understand

Creating Multiple Instances

Create as many instances from a class as you need. Each instance is separate with its own set of attributes:

my_lion = Lion('Mufasa', 12)
your_lion = Lion('Simba', 8)

print(f"My lion's name is {my_lion.name}.")
print(f"My lion is {my_lion.age} years old.")
my_lion.roar()

print(f"\nYour lion's name is {your_lion.name}.")
print(f"Your lion is {your_lion.age} years old.")
your_lion.pace()

Expected Output:

My lion's name is Mufasa.
My lion is 12 years old.
Mufasa lets out a mighty roar!

Your lion's name is Simba.
Your lion is 8 years old.
Simba paces back and forth nervously.

Key Point:

  • Each lion is a separate instance with its own attributes
  • Both lions have the same methods and can perform the same actions
  • Even with the same name and age, each instance would be separate
  • Create as many instances as needed by assigning each to a unique variable

Working with Classes and Instances

Most of your programming time involves working with instances created from classes.

Common tasks:

  • Modify attributes of an instance
  • Update attributes directly or through methods
  • Validate changes before updating

Modifying Attribute Values

You can change an attribute’s value in three ways:

  1. Change directly through an instance — access the attribute directly
  2. Set through a method — call a method that handles the update
  3. Increment through a method — add a certain amount to the value

Approach 1: Modify Directly

Use dot notation to change an attribute’s value directly:

first_lion = Lion('Scar', 12, 95)
print(first_lion.get_descriptive_name())

# Modify hunger level directly
first_lion.hunger_level = 8
first_lion.check_hunger()

Expected Output:

Scar The Lion - Age 12, Strength 95
This lion's hunger level is 8.

How it works:

  • Use dot notation: instance.attribute = new_value
  • Python finds the attribute and sets it directly
  • Simple, but sometimes you want more control — that’s where methods come in

Approach 2: Modify Through a Method

Write a method to update attributes. This lets you add validation and control:

Benefits of using a method:

  • Centralise the update logic
  • Add validation and error checking
  • Make code easier to maintain

Add validation: Prevent invalid updates (e.g., hunger level above 10):

class Lion:
    """A model of a lion in the den."""
    
    def __init__(self, name, age, strength):
        self.name = name
        self.age = age
        self.strength = strength
        self.hunger_level = 0
    
    def get_descriptive_name(self):
        long_name = f"{self.name} the Lion - Age {self.age}, Strength {self.strength}"
        return long_name.title()
    
    def check_hunger(self):
        print(f"This lion's hunger level is {self.hunger_level}.")
    
    def set_hunger(self, level):
        """
        Set the hunger level to the given value.
        Reject the change if it exceeds maximum hunger (10).
        """
        if level <= 10:
            self.hunger_level = level
        else:
            print("Hunger level cannot exceed 10!")

first_lion = Lion('Scar', 12, 95)
first_lion.set_hunger(8)
first_lion.check_hunger()

# Attempt to set hunger too high
first_lion.set_hunger(12)

Expected Output:

This lion's hunger level is 8.
Hunger level cannot exceed 10!

Validation in action:

  • Method checks if the new level is valid (max 10)
  • Only updates if level <= 10
  • Prints a warning if someone tries an invalid update
  • This adds business logic and prevents mistakes

Approach 3: Increment Through a Method

Increment an attribute by adjusting it relative to its current value, rather than setting a fixed new value.

Scenario: A lion gets fed — feeding reduces its hunger level:

class Lion:
    """A model of a lion in the den."""
    
    def __init__(self, name, age, strength):
        self.name = name
        self.age = age
        self.strength = strength
        self.hunger_level = 0
    
    def get_descriptive_name(self):
        long_name = f"{self.name} the Lion - Age {self.age}, Strength {self.strength}"
        return long_name.title()
    
    def check_hunger(self):
        print(f"This lion's hunger level is {self.hunger_level}.")
    
    def set_hunger(self, level):
        if level <= 10:
            self.hunger_level = level
        else:
            print("Hunger level cannot exceed 10!")
    
    def feed_lion(self, meals):
        """Reduce hunger by the given number of meals (minimum 0)."""
        self.hunger_level = max(0, self.hunger_level - meals)

# Create a new lion
second_lion = Lion('Scar', 8, 95)
print(second_lion.get_descriptive_name())

# Set initial hunger
second_lion.set_hunger(3)
second_lion.check_hunger()

# Feed the lion
second_lion.feed_lion(2)
second_lion.check_hunger()

Expected Output:

Scar The Lion - Age 8, Strength 95
This lion's hunger level is 3.
This lion's hunger level is 1.

How incrementing works:

  • max(0, self.hunger_level - meals) reduces hunger but prevents it from going below 0
  • Useful when adjusting a value relative to its current state rather than replacing it entirely
  • Can combine with validation to also prevent exceeding the upper limit

Key Point: Methods give you control over how and when attributes are updated. Use them to validate data, add business logic, and make your code more maintainable.

Setter and Getter Methods

  • Getter methods return the value of an attribute (e.g., check_hunger())
  • Setter methods update the value of an attribute (e.g., set_hunger())

Getters and setters provide a controlled interface to access and modify attributes, allowing you to add validation and maintain encapsulation. In contrast to other programming languages that often require explicit getter and setter methods, Python allows direct attribute access instead of getters and setters, but using methods can still be beneficial for validation and control.

Solution

from lion2 import Lion

class Den:
    def __init__(self):
        self.lions = []
        self.is_sealed = True
        
    def add_lion(self, lion_object):
        if self.is_sealed:
            print('The den is sealed. Cannot add a lion.')
            return
        self.lions.append(lion_object)
        
    def get_hunger_report(self):
        if self.is_sealed:
            print('The den is sealed. Cannot check the lions.')
            return

        # 1. Calculate stats
        total_hunger = sum(lion.hunger_level for lion in self.lions)
        
        # Avoid division by zero if den is empty!
        count = len(self.lions)
        avg_hunger = total_hunger / count if count > 0 else 0
        
        # 2. Return the dictionary report
        return {
            "total_threat": total_hunger,
            "average_hunger": avg_hunger,
            "lion_count": count
        }

    def open(self):
        self.is_sealed = False
    
    def close(self):
        self.is_sealed = True

# 1. Create Lions

leo.set_hunger(7)
sultan = Lion("Sultan", 8, 23)
sultan.set_hunger(4)

babylon_den.open()

babylon_den.add_lion(sultan)

Inheritance

Don’t start from scratch! Use inheritance when writing a specialized version of an existing class.

Key concepts:

  • Parent class: The original class (also called superclass)
  • Child class: A new class that inherits from the parent (also called subclass)
  • Child classes inherit all attributes and methods from the parent
  • Child classes can define new attributes and methods
  • Child classes can override parent methods

Example: Minister (Child) from Citizen (Parent)

Scenario: A Minister is a special type of Citizen with the ability to suggest royal decrees.

A minister has everything a regular citizen has.

class Citizen:
    """Represent a citizen of the kingdom."""
    
    def __init__(self, name, age):
        """Initialize attributes to describe a citizen."""
        self.name = name
        self.age = age
    
    def get_description(self):
        """Return a formatted description of the citizen."""
        return f"{self.name}, age {self.age}"
    
    def pay_taxes(self):
        """Pay taxes to the kingdom."""
        print(f"{self.name} pays taxes to the kingdom.")


class Minister(Citizen):
    """A minister is a special citizen in the royal court."""
    
    def __init__(self, name, age):
        """Initialize attributes of the parent class."""
        super().__init__(name, age)

# Create a minister instance
minister = Minister('Daniel', 40)
print(minister.get_description())

Expected Output:

Daniel, age 40

How inheritance works:

  1. Parent class appears first — Citizen is defined before Minister
  2. Child class inherits — Minister(Citizen) inherits from Citizen
  3. Call parent __init__() — super().__init__() calls the parent’s __init__() method
  4. super() is a special function that accesses parent class methods
  5. All parent attributes available — Minister gets name and age from Citizen
  6. All parent methods available — Minister can use get_description() and pay_taxes()

Defining Attributes and Methods for the Child Class

A Minister is a special type of Citizen, so they have everything a citizen has, plus additional features specific to ministers.

Add new attributes and methods to differentiate the child class from the parent.

Don’t overcomplicate the parent class — only add attributes/methods specific to the child class to the child.

class Citizen:
    """Represent a citizen of the kingdom."""
    
    def __init__(self, name, age):
        """Initialize attributes to describe a citizen."""
        self.name = name
        self.age = age
    
    def get_description(self):
        """Return a formatted description of the citizen."""
        return f"{self.name}, age {self.age}"
    
    def pay_taxes(self):
        """Pay taxes to the kingdom."""
        print(f"{self.name} pays taxes to the kingdom.")

class Minister(Citizen):
    """A minister is a special citizen in the royal court."""
    
    def __init__(self, name, age, respect_from_king=50):
        """
        Initialize attributes of the parent class.
        Then initialize attributes specific to a minister.
        """
        super().__init__(name, age)
        self.respect_from_king = respect_from_king
    
    def suggest_law(self):
        """Suggest a new law to the king."""
        print(f"{self.name} suggests a new law. The king's respect: {self.respect_from_king}/100")

minister = Minister('Daniel', 40)
print(minister.get_description())
minister.suggest_law()

Expected Output:

Daniel, age 40
Daniel suggests a new law. The king's respect: 50/100

Key Points:

  • The Minister class inherits from Citizen, gaining all parent attributes and methods
  • super().__init__() calls the parent’s __init__() method first
  • After the parent initialization, we can add child-specific attributes like respect_from_king
  • New methods like suggest_law() are specific to the child class
  • The child class has access to parent methods: minister.get_description() works because Minister inherited it

Overriding Methods of the Parent Class

When the behavior of a child class should differ from its parent class, you can override the parent method.

  • Define a method in the child class with the same name as a parent method
  • The child’s version will be used instead of the parent’s version
  • This allows specialized behavior while maintaining the inheritance structure
class Citizen:
    """Represent a citizen of the kingdom."""
    
    def __init__(self, name, age):
        """Initialize attributes to describe a citizen."""
        self.name = name
        self.age = age
    
    def get_description(self):
        """Return a formatted description of the citizen."""
        return f"{self.name}, age {self.age}"
    
    def speak_to_the_king(self):
        """Speak to the king."""
        print(f"{self.name} speaks to the king. But the king does not listen.")

class Minister(Citizen):
    """A minister is a special citizen in the royal court."""
    
    def __init__(self, name, age, respect_from_king=50):
        """
        Initialize attributes of the parent class.
        Then initialize attributes specific to a minister.
        """
        super().__init__(name, age)
        self.respect_from_king = respect_from_king
    
    def speak_to_the_king(self):
        """Override speak_to_the_king with authority."""
        if self.respect_from_king >= 90:
            print(f"{self.name} speaks to the king. He agrees.")
        else:
            print(f"{self.name} speaks to the king. He will think about it.")

# Test both behaviors
common_citizen = Citizen("Marcus", 35)
common_citizen.speak_to_the_king()

minister = Minister("Daniel", 40, 100)
minister.speak_to_the_king()

Expected Output:

Marcus speaks to the king. But the king does not listen.
Daniel speaks to the king. He agrees.

What Happened:

  • Both Citizen and Minister have a speak_to_the_king() method
  • When we call speak_to_the_king() on a Minister instance, Python uses the child’s version
  • The child’s version can completely replace or extend the parent’s behavior
  • This is powerful because different types of citizens can behave differently while sharing common structure

Instances as Attributes

Instead of putting everything in one class, you can create smaller classes and use them as attributes in larger classes.

  • This approach is called composition
  • It helps organise complex objects into manageable pieces
  • A kingdom might have a king (who is also a citizen) as an attribute
class King(Citizen):
    """A king is a special citizen who rules the kingdom."""
    
    def __init__(self, name, age, reign_years):
        """Initialize attributes of the parent class and king-specific attributes."""
        super().__init__(name, age)
        self.reign_years = reign_years
    
    def decree(self, law_description):
        """The king issues a decree."""
        print(f"King {self.name} decrees: {law_description}")

class Kingdom:
    """Represent a kingdom with a ruling king."""
    
    def __init__(self, kingdom_name, king_name, king_age, reign_years):
        self.kingdom_name = kingdom_name
        # King is now a separate object (an instance of the King class)
        self.king = King(king_name, king_age, reign_years)
    
    def get_status(self):
        """Display the kingdom's status."""
        return f"{self.kingdom_name} is ruled by {self.king.get_description()}"

# Create a kingdom with a king
babylon = Kingdom("Babylon", "Darius", 52, 15)

# Access the kingdom's status
print(babylon.get_status())

# Access the king's methods through the kingdom
babylon.king.decree("All must obey the law of the land!")

Expected Output:

Babylon is ruled by Darius, age 52
King Darius decrees: All must obey the law of the land!

Key Concepts:

  • King is a child class that inherits from Citizen
  • Kingdom is a separate class that creates a King instance as an attribute: self.king = King(...)
  • We access the king’s methods through the kingdom: babylon.king.decree()
  • We access the king’s attributes through the kingdom: babylon.king.get_description()
  • This demonstrates composition: a Kingdom contains a King, and the King is a special Citizen

Modeling Real-World Objects

When designing classes, think about structure and relationships:

Questions to ask:

  • What are the attributes of this object?
  • What can this object do (methods)?
  • Will this object have specialized variations (child classes)?
  • Are there smaller objects within this object (composition)?

Design Philosophy:

  • Start simple — don’t add everything at once
  • It’s okay (even good!) to restructure code as you learn more
  • Multiple designs can be correct — it depends on your needs
  • You might use inheritance for some relationships and composition for others

Example from our work:

  • We created a basic Citizen class
  • Then added Minister as a specialized version (inheritance) that can suggest laws
  • Then added King as another specialized Citizen (inheritance) who can issue decrees
  • Then created Kingdom that contains a King instance (composition)
  • This gives us flexible code where each class has a clear purpose

Importing Classes

As your classes grow more complex, keep your files clean by storing classes in separate modules.

Key Benefits:

  • Keep individual files short and readable
  • Organise related classes together
  • Reuse classes across multiple programs
  • Separate logic from main program

Importing a Single Class

Store a class in its own module, then import it where needed.

File: citizen.py — Contains just the Citizen class with a module-level docstring

"""Classes for representing citizens of the kingdom."""

class Citizen:
    """Represent a citizen of the kingdom."""
    
    def __init__(self, name, age):
        """Initialize attributes to describe a citizen."""
        self.name = name
        self.age = age
    
    def get_description(self):
        """Return a formatted description of the citizen."""
        return f"{self.name}, age {self.age}"
    
    def pay_taxes(self):
        """Pay taxes to the kingdom."""
        print(f"{self.name} pays taxes to the kingdom.")

File: main.py

from citizen import Citizen

# Import Citizen and use it
marcus = Citizen('Marcus', 35)
print(marcus.get_description())
marcus.pay_taxes()

Expected Output:

Marcus, age 35
Marcus pays taxes to the kingdom.

Key Points:

  • from citizen import Citizen imports just the Citizen class from the citizen module
  • Your main program file stays clean and focused
  • The class logic is isolated in its own module
  • Once the module works, you can use it without further changes

Storing Multiple Classes in a Module

Store related classes together in one module.

File: kingdom.py — Contains Citizen, Minister, and King classes

"""A set of classes used to represent citizens in the kingdom."""

class Citizen:
    """Represent a citizen of the kingdom."""
    def __init__(self, name, age):
        self.name = name
        self.age = age
    def get_description(self):
        return f"{self.name}, age {self.age}"


class Minister(Citizen):
    """A minister is a special citizen in the royal court."""
    def __init__(self, name, age):
        super().__init__(name, age)
        self.respect_from_king = 50
    def suggest_law(self):
        print(f"{self.name} suggests a new law.")


class King(Citizen):
    """A king is a special citizen who rules the kingdom."""
    def __init__(self, name, age, reign_years):
        super().__init__(name, age)
        self.reign_years = reign_years
    def decree(self, law_description):
        print(f"King {self.name} decrees: {law_description}")

Importing Multiple Classes from a Module

Import several related classes with a single statement.

from kingdom import Citizen, Minister, King

# Create different types of citizens
marcus = Citizen("Marcus", 35)
print(marcus.get_description())

daniel = Minister("Daniel", 40)
daniel.suggest_law()

darius = King("Darius", 52, 15)
darius.decree("All must follow the law!")

Expected Output:

Marcus, age 35
Daniel suggests a new law.
King Darius decrees: All must follow the law!

Syntax:

  • Separate class names with commas: from module import Class1, Class2, Class3
  • Import only what you need
  • Makes it clear what classes your program uses

Importing an Entire Module

Use dot notation to access classes from a module.

import kingdom

# Access classes using dot notation
marcus = kingdom.Citizen("Marcus", 35)
print(marcus.get_description())

daniel = kingdom.Minister("Daniel", 40)
daniel.suggest_law()

darius = kingdom.King("Darius", 52, 15)
darius.decree("All must follow the law!")

Advantages:

  • Clear where each class comes from: kingdom.King
  • Avoids naming conflicts
  • Easy to read and understand module relationships

Using Aliases

Give classes or modules shorter names for convenience.

Alias a class:

from kingdom import King as Ruler

# Use the alias
darius = Ruler("Darius", 52, 15)
darius.decree("All must follow the law!")

Alias a module:

import kingdom as k

# Use the module alias
darius = k.King("Darius", 52, 15)
darius.decree("All must follow the law!")

Styling Classes

Follow Python conventions for consistent, professional code.

Class naming:

  • Use CamelCase for class names: Citizen, Minister, King
  • Capitalise the first letter of each word
  • No underscores in class names

Module and instance naming:

  • Use lowercase with underscores for module names: kingdom.py, citizen.py
  • Use lowercase with underscores for variable names: my_citizen, darius_king

Docstrings:

  • Include a docstring immediately after class definition
  • Describe what the class represents
  • Include module-level docstring describing what classes are in the module

Spacing:

  • One blank line between methods in a class
  • Two blank lines between classes in a module

What you get:

How to use:

The random Module

Generate random numbers and make random selections.

from random import randint, choice

# Roll a die (random number between 1 and 6)
die_roll = randint(1, 6)
print(f"Lion's agitation level: {die_roll}/10")

# Randomly select a citizen from the kingdom
citizens = ['Marcus', 'Daniel', 'Shadrach', 'Meshach', 'Abednego']
chosen_citizen = choice(citizens)
print(f"Selected for the den: {chosen_citizen}")

Expected Output (values vary — output is random):

Lion's agitation level: 4/10
Selected for the den: Daniel

Key Functions:

  • randint(a, b) — Returns random integer between a and b (inclusive)
  • choice(sequence) — Returns random element from a list or tuple

Use Case:

  • Simulations and games
  • Randomly selecting data samples
  • Modelling unpredictable situations

The sys Module

Access system-specific parameters and functions.

import sys

# Get Python version
print(f"Python version: {sys.version}")

# Get command line arguments (useful for passing data to programs)
print(f"Program name: {sys.argv[0]}")

# Access platform information
print(f"Platform: {sys.platform}")

Expected Output (values depend on your system):

Python version: 3.12.0 (...)
Program name: main.py
Platform: linux

Key Attributes:

  • sys.version — Python version information
  • sys.argv — Command-line arguments passed to the program
  • sys.platform — Information about the operating system
  • sys.exit() — Exit the program with a status code

Use Case:

  • Reading user input from command line
  • Building cross-platform programs
  • Debugging and getting system information

The time Module

Work with dates, times, and delays.

import time

# Get current time
current_time = time.time()
print(f"Current time (seconds since epoch): {current_time}")

# Format current date and time
formatted_time = time.strftime("%Y-%m-%d %H:%M:%S")
print(f"Formatted time: {formatted_time}")

# Create a delay (sleep for 1 second)
print("Daniel enters the den...")
time.sleep(1)
print("The lions are peaceful.")

Expected Output (time values depend on when you run it):

Current time (seconds since epoch): 1746350000.0
Formatted time: 2026-05-04 10:30:00
Daniel enters the den...
The lions are peaceful.

Key Functions:

  • time.time() — Seconds since January 1, 1970 (epoch)
  • time.sleep(seconds) — Pause execution for specified seconds
  • time.strftime(format) — Format current time as a readable string
  • time.localtime() — Get current date/time as a structured time object

Use Case:

  • Measuring performance and execution time
  • Adding delays to programs
  • Displaying formatted dates and times
  • Creating time-based events
